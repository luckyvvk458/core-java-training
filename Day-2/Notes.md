# Day 2 - Object-Oriented Programming (OOP) Concepts

## Objective

Understand the fundamentals of Object-Oriented Programming and learn how real-world entities can be represented using software objects.

---

## What is OOP?

Object-Oriented Programming (OOP) is a programming paradigm that organizes software around objects rather than functions.

OOP helps in building:

* Reusable code
* Maintainable applications
* Scalable systems
* Real-world models

Java is primarily an Object-Oriented Programming language.

---

## Class and Object

### Class

A class is a blueprint or template used to create objects.

Example:

```text
Car Blueprint
```

The blueprint describes:

* Color
* Brand
* Speed

but it is not an actual car.

---

### Object

An object is a real instance created from a class.

Example:

```text
Car 1
Car 2
Car 3
```

All are objects created from the same blueprint.

---

## Real-Life Example

### Class

```text
Student
```

Attributes:

```text
id
name
marks
```

---

### Objects

```text
Student 1
Student 2
Student 3
```

Each student has different values but belongs to the same class.

## Four Pillars of OOP

1. Encapsulation
2. Inheritance
3. Polymorphism
4. Abstraction

---

## Encapsulation

### Definition

Binding data and methods together into a single unit.

Encapsulation helps in:

* Data Hiding
* Security
* Better Maintenance

Example:

```text
ATM Machine
```

Users interact with buttons.

Internal implementation is hidden.

---

### Benefits

* Protects data
* Prevents unauthorized access
* Improves maintainability

---

## Inheritance

### Definition

Inheritance allows one class to acquire properties and behaviors of another class.

Example:

```text
Animal
   |
   └── Dog
```

Dog can inherit common features from Animal.

---

### Benefits

* Code Reusability
* Reduced Duplication
* Easy Maintenance

---

## Polymorphism

### Definition

One action can behave differently depending on the object.

Example:

```text
Vehicle.start()
```

For a Bike:

```text
Kick Start
```

For a Car:

```text
Self Start
```

Same action.

Different behavior.

---

### Benefits

* Flexibility
* Extensibility
* Cleaner Design

---

## Abstraction

### Definition

Showing only essential details and hiding implementation details.

Example:

```text
Car
```

A driver only knows:

* Steering
* Accelerator
* Brake

The engine internals are hidden.

---

### Benefits

* Simplicity
* Reduced Complexity
* Better Security

---

## Veera Simham Story

To understand OOP concepts, we used the Veera Simham story.

### Encapsulation

Veera Simham protects his valuables inside the fort.

Data is protected from direct access.

---

### Inheritance

The prince inherits qualities and responsibilities from Veera Simham.

Just like a child class inherits from a parent class.

---

### Polymorphism

Veera Simham behaves differently in different situations:

* King in the kingdom
* Father at home
* Warrior in battle

One person.

Multiple behaviors.

---

### Abstraction

People see Veera Simham as a powerful king.

The complex strategies behind his decisions remain hidden.

Only important information is exposed.

---

## Key Takeaways

* A class is a blueprint.
* An object is an instance of a class.
* OOP helps model real-world entities.
* Encapsulation protects data.
* Inheritance promotes reuse.
* Polymorphism provides flexibility.
* Abstraction hides complexity.
